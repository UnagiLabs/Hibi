Hibi / Family memories, remembered.

Hibi の全体像をつかむ説明書

Hibi は、チャットに送った育児ログや写真を、DB・MemWal・Walrus・Sui に整理して保存し、あとから Web URL で見返せる Family Memory Agent です。

1. まず一言でいうと

ユーザー体験

家族がチャットに「ミルク120ml飲んだ」「この写真、はじめて寝返りした！」のように送るだけで、Hibi が記録・保存・検索・閲覧URL発行まで行います。

技術的な役割

Hibi API が中心になり、ルールによる下書き分類、LLMによる確認・補正、DB保存、MemWal remember/recall、Walrus upload、Sui pointer/hash 記録、Web表示用データ取得を担当します。

2. 全体の流れ

Chat OpenClaw などから自然文や写真が送られる
Plugin チャット入力を Hibi API に渡す薄い入口
Hibi API 分類、保存、URL発行などの司令塔
DB ログ、思い出、写真情報、閲覧URLを永続化
MemWal あとからAIが思い出せる長期記憶
Walrus / Sui 写真・アルバム保存、hashやpointerの検証
Web URLを開いて育児ログやアルバムを見る

3. 部品ごとの役割

OpenClaw Plugin

チャットアプリ側の入口です。できるだけ薄くして、入力を Hibi API に渡し、返ってきた返信文とURLを表示します。

Hibi API

Hibi の中心です。POST /api/messages などを受け取り、分類、DB保存、外部サービス連携、URL発行を行います。

DB

再起動後も残すための保存先です。CareLog、MemoryItem、MediaAsset、Album、MemoryViewSession を保存します。

MemWal

AIの長期記憶です。「はじめて寝返りした」などの意味を保存し、「最近できるようになったことは？」で recall します。

Walrus

写真や生成したアルバムmanifestなど、長く残したい artifact を保存します。ハッカソンでは実接続が重要です。

Sui

写真本文や家族情報を直接載せる場所ではありません。Walrus blob の pointer や hash など、検証用metadataを記録します。

Web App

チャットで返された /v/:viewId を開いて、育児ログ、写真、月次アルバム、保存状態を閲覧する画面です。

Core Package

共通型、intent分類、MemWal/Walrus/Sui client、workflow helper などを置く共有パッケージです。

Contracts

Sui Move contract を置く場所です。MVPでは後半で、pointer/hash 表示まで縮小してもよい領域です。

4. 代表的なユーザーフロー

育児ログ

ユーザーが「ミルク120ml飲んだ」と送ると、Hibi API が育児ログとして分類し、DBに保存し、今日のログURLを返します。

POST /api/messages
{
  "text": "ミルク120ml飲んだ"
}

-> CareLog 保存
-> /v/:viewId 発行
-> Webで今日のログを表示

写真と思い出

ユーザーが写真と「はじめて寝返りした！」を送ると、写真は Walrus、コメントはDBとMemWalに保存します。

POST /api/photos
photo.jpg + comment

-> Walrus upload
-> MediaAsset 保存
-> MemoryItem 保存
-> MemWal remember

5. 最初に作るべきもの

いきなり MemWal / Walrus / Sui から作るのではなく、まず「チャット入力がDBに残り、URLでWeb表示できる」土台を作るのが安全です。

Step 1

API skeleton

GET /api/health と POST /api/messages を作ります。ここで Hibi API の形を決めます。

Step 2

DB schema

User、Family、MemorySpace、CareLog、MemoryItem、MediaAsset、Album、MemoryViewSession を用意します。

Step 3

rule-based intent

ルールで「ミルク」「母乳」「おっぱい」「寝た」「今起きた」「うんこ」「体温」などを構造化し、LLM validatorで確認・補正します。

Step 4

care log view

チャット返信に含まれるURLを開くと、DBに保存された今日の育児ログが表示されるようにします。

最初の完成ライン ミルク120ml飲んだ を送るとDBに保存され、返ってきたURLを開くとWebで見える状態。

6. 今は後回しでよいもの

OpenClaw Plugin

APIが固まってから作る方が楽です。最初は curl や簡単なテストUIから Hibi API を叩けば十分です。

本格的なSui contract

ハッカソンMVPでは、まず DB 上で hash/pointer を見せるところまで縮小できます。ただし Walrus と MemWal の実接続は重要です。

高度なAI分類

LLMは最初から確認役として使います。ただしLLM障害時も、高confidenceのルール結果なら保存workflowを継続できるようにします。

複雑なWeb入力画面

Hibi の中心はチャット入力です。Webはまず閲覧専用のアルバム、育児ログ、保存状態に集中します。

7. 用語ミニ辞典

Intent

ユーザー入力の目的です。例: care_log、photo_memory、ask_memory。

CareLog

ミルク、睡眠、排泄、体温などの育児ログです。

MemoryItem

写真コメントや日々の出来事など、あとから思い出として見返す本文です。

MediaAsset

写真などのメディア情報です。Walrus blob ID もここに紐づきます。

AlbumManifest

月次アルバムを表示するためのJSONです。Walrusに保存するartifact候補です。

MemoryViewSession

チャット要求に応じて発行される閲覧URLです。Album本体とは分けて管理します。
